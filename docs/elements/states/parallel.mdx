# Parallel Element

The `<parallel>` element enables concurrent execution of multiple states:

import { DynamicCodeBlock } from 'fumadocs-ui/components/dynamic-codeblock';
import { TypeTable } from 'fumadocs-ui/components/type-table';

<DynamicCodeBlock lang="jsx"  code={`<parallel id="concurrent">
  <state id="process1">
    <transition target="done1" />
  </state>
  
  <state id="process2">
    <transition target="done2" />
  </state>
</parallel>`} />

## Allowed Children

- `onentry`: Entry actions
- `onexit`: Exit actions
- `transition`: State transitions
- `state`: Child states
- `history`: History pseudo-states
- `datamodel`: Data model declarations
- `invoke`: Service invocations

## Usage Notes

- All child states execute concurrently
- Useful for independent processes
- Can share data through parent scope
- Transitions can span parallel regions

## Best Practices

1. **State Independence**
   - Keep states autonomous
   - Minimize state dependencies
   - Handle shared resources
   - Manage concurrency

2. **Resource Management**
   - Track resource usage
   - Clean up properly
   - Handle resource conflicts
   - Monitor resource state

3. **Error Handling**
   - Handle partial failures
   - Provide recovery paths
   - Coordinate error states
   - Log error conditions

4. **Performance**
   - Balance parallel load
   - Monitor execution
   - Handle timeouts
   - Consider resource limits
