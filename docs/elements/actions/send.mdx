# Send Element

The `<send>` element sends an event to an external system:

<DynamicCodeBlock lang="jsx" code={`<send 
  event="notification"
  target="external-service"
  type="http"
/>`} />

## Examples

### HTTP Request

<DynamicCodeBlock lang="jsx" code={`<send
  type="http"
  target="https://api.example.com/endpoint"
  event="postData"
  namelist="state.userData"
/>`} />

### Delayed Event

<DynamicCodeBlock lang="jsx" code={`<send
  event="reminder"
  delay="5s"
  id="reminderEvent"
/>`} />

### Dynamic Target

<DynamicCodeBlock lang="jsx" code={`<send
  event="notification"
  targetexpr={({state}) => state.serviceEndpoint}
  type="websocket"
/>`} />

## Usage Notes

- Use for external system communication
- Can be delayed with the delay attribute
- Supports various protocols via type
- Can include data model values

## Best Practices

1. **Event Configuration**
   - Use appropriate event types
   - Set reasonable delays
   - Include necessary data
   - Handle timeouts

2. **Target Management**
   - Validate target endpoints
   - Use secure protocols
   - Handle connection failures
   - Monitor response times

3. **Data Handling**
   - Validate data before sending
   - Format data appropriately
   - Handle sensitive information
   - Clean up after sending

4. **Error Handling**
   - Handle network errors
   - Provide timeout handling
   - Log send failures
   - Implement retries
