# Raise Element

The `<raise>` element raises an internal event:

<DynamicCodeBlock lang="jsx" code={`<raise event="success" />`} />

## Usage Notes

- Use for internal state machine communication
- Events are processed immediately
- Can trigger state transitions
- Useful for workflow control

## Best Practices

1. **Event Naming**
   - Use clear, descriptive event names
   - Follow consistent naming conventions
   - Document event purposes
   - Consider using namespaces for organization

2. **Event Flow**
   - Plan event sequences carefully
   - Avoid circular event chains
   - Consider event priority
   - Document event flow

3. **State Management**
   - Coordinate with state transitions
   - Handle all possible event outcomes
   - Clean up after event processing
   - Consider event history

4. **Error Handling**
   - Handle unexpected events
   - Provide error events
   - Log event processing
   - Include fallback behavior
